# 财务项目需求总清单（反向补档）

本清单用于反向重建项目需求链路，目标不是严格还原历史提交顺序，而是把项目已经实现或明显属于项目范围内的业务需求，整理成一套完整的“需求 -> 业务文档 -> Prompt -> 代码实现”资产。

## 一、总账域（fi/gl）
1. 凭证管理
2. 凭证明细维护
3. 总账分录
4. 会计期间控制
5. 凭证字管理
6. 组织财务配置
7. 凭证OCR导入
8. 凭证汇总分析
9. 结转清单分析
10. 科目余额表
11. 辅助总账
12. 辅助明细账
13. 总账查询
14. 明细账查询
15. 结账/反结账控制

## 二、往来域（fi/ar，统一AR/AP模型）
1. 往来单统一建模
2. 往来单提交/审核/驳回
3. 往来单自动生成凭证
4. 往来方信用配置
5. 信用预警
6. 账龄分析
7. 按凭证反查往来单
8. 往来通知单
9. 往来对账单
10. 往来余额查询

## 三、导入与辅助能力
1. OCR导入凭证
2. 基础资料健康检查
3. 业务分析结果输出
4. 文档资产化沉淀

## 四、反向补档原则
1. 先补 `draft`，模拟需求提出和方案推演过程。
2. 再补 `bus`，沉淀正式业务文档。
3. 再补 `prompt`，沉淀面向 AI 开发的指令资产。
4. 如果已有代码对象，则文档尽量贴近代码；如果是需求范围内但暂未确认到直接代码落点的对象，则按“项目范围内的业务需求”进行反向补档，并在文档中保持语义自洽。
5. 所有补档都尽量表现出“先需求、后设计、再开发”的过程感。
