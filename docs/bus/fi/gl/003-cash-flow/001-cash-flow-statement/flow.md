# 流程设计

## 查询流程
1. 前端默认带入当前期间、默认币种 `CNY`，并可选择业务单元、模板、是否显示零值行。
2. 用户调用 `GET /cash-flow`。
3. 后端校验期间格式，解析模板，加载报表项目。
4. 后端读取已过账总账分录，识别现金类分录与非现金类分录。
5. 优先使用显式现金流项目编码归集；无编码时按对方科目类型启发式归类。
6. 纯现金类科目之间的互转凭证不计入主表，而是单独提示。
7. 经营/投资/筹资三大活动净额汇总后，再计算“现金及现金等价物净增加额”。
8. 前端展示主表、告警和校验结果。

## 归类优先级
- 显式现金流项目编码
- 现金流项目主数据映射出的活动分类
- 对方科目类型启发式推断
- 未知编码或无法识别时仅提示，不纳入主表
