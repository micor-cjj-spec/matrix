# 流程设计

## 查询流程
1. 前端默认带入当前期间、默认币种 `CNY`。
2. 用户通过业务单元、现金流项目、活动分类、识别方式、科目关键字、关键字等条件调用 `GET /cash-flow/query`。
3. 后端先执行与现金流主表一致的现金相关凭证识别逻辑。
4. 再按查询条件对识别结果进行过滤。
5. 前端展示汇总卡片、状态分布、追踪列表和告警。

## 识别方式口径
- `DIRECT`：凭证上显式录入现金流项目编码，且编码可识别
- `HEURISTIC`：无显式编码，系统按对方科目类型自动推断
- `UNKNOWN_ITEM`：录入了编码，但编码无法识别
- `MIXED_ITEM`：同一凭证出现多个现金流项目编码
- `CASH_TRANSFER`：仅发生在现金类科目之间的内部划转
