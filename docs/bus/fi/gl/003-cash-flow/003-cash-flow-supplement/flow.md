# 流程设计

## 查询流程
1. 前端默认带入当前期间、默认币种 `CNY`。
2. 用户通过业务单元和期间调用 `GET /cash-flow/supplement`。
3. 后端先执行与现金流主表一致的现金相关凭证识别逻辑。
4. 在识别结果上生成补录检查项、分类分布和待复核凭证清单。
5. 前端展示汇总卡片、任务清单、分类分布、待处理凭证和告警。

## 补录输出逻辑
- `tasks`：输出主数据、直接标记覆盖率、未知编码、多编码、现金划转、组织隔离等检查项。
- `categories`：按分类桶汇总分布。
- `pendingVouchers`：只保留非 `DIRECT` 的记录，按问题优先级排序。
