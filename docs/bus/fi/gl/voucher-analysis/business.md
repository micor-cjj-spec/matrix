# 凭证分析业务说明

## 1. 业务名称
凭证分析（Voucher Analysis）

## 2. 业务背景
凭证分析用于在凭证基础流程之上提供汇总统计和结转清单分析能力。当前代码已支持按日期、状态、摘要关键字做汇总查询，也支持结合组织财务参数、会计期间、凭证字、基础资料健康检查输出结转清单。

## 3. 业务目标
- 提供凭证汇总统计能力
- 提供期末结转前的辅助检查清单
- 复用组织财务参数、会计期间、默认凭证字等已有配置

## 4. 单据定位
凭证分析属于 `fi/gl` 域的分析型功能对象，不直接产生账务数据，而是基于现有凭证和配置输出分析结果。

## 5. 核心业务内容
- 按日期、状态、摘要关键字汇总凭证数量和金额
- 识别结转类凭证并生成结转清单
- 读取组织财务配置、会计期间、默认凭证字和基础资料健康检查结果
- 输出警告信息和待处理事项

## 6. 上下游关系
### 上游
- 凭证数据
- 组织财务配置
- 会计期间
- 凭证字
- 基础资料健康检查结果

### 下游
- 结账前检查
- 运营分析
- 财务排查与辅助判断

## 7. 关键控制点
- 筛选参数必须格式正确
- 汇总分析和结转清单分开处理
- 结转清单不是正式过账动作，只提供辅助判断和提示
