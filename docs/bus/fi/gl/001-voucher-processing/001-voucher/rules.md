# 业务规则

## 主表规则
- 凭证参数不能为空。
- 凭证日期不能为空。
- 凭证摘要不能为空。
- 保存草稿时校验会计期间是否已关账；命中环境变量 `FI_CLOSED_PERIODS` 时禁止操作。
- 凭证号允许前端传入，但必须唯一；为空时后端自动生成。
- 只有 `DRAFT / REJECTED` 状态允许修改。
- 只有 `DRAFT` 状态允许删除。

## 分录规则
- 凭证至少需要 2 条分录才能提交/过账。
- 每行科目编码不能为空。
- 借贷金额不能小于 0。
- 借贷金额不能同时为 0。
- 借贷金额不能同时大于 0。
- 严格校验场景下，借方合计必须等于贷方合计。
- 保存分录时仅允许 `DRAFT / REJECTED` 状态维护。

## 状态流转规则
- 只有 `DRAFT / REJECTED` 可提交。
- 只有 `SUBMITTED` 可审核。
- 只有 `SUBMITTED` 可驳回。
- 只有 `AUDITED` 可过账。
- 只有 `POSTED` 可冲销。

## 过账规则
- 过账前会再次校验分录与借贷平衡。
- 过账会先删除已存在的总账分录，再重新生成，保证幂等。
- 过账完成后写入 `fpostedBy / fpostedTime`。

## 冲销规则
- 冲销会生成新的已过账凭证和对应总账分录。
- 原凭证状态改为 `REVERSED`。
- 新凭证摘要、分录摘要均追加“冲销:”。
