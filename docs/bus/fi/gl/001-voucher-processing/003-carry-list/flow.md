# 流程设计

## 页面流程
1. 页面初始化先加载业务单元列表。
2. 默认取当前月份作为期间，并优先选中第一个业务单元。
3. 前端调用 `GET /voucher/carry-list`。
4. 后端解析期间来源，并依次检查：组织财务参数、会计期间、默认凭证字、基础资料健康度。
5. 后端再加载当前期间的凭证数据，并识别结转相关凭证。
6. 前端展示元信息、检查任务清单、结转相关凭证清单。

## 期间解析流程
- 先取页面输入 `period`
- 若页面输入无效，则回退组织财务参数 `fcurrentPeriod`
- 若组织参数也无效，则回退系统当前月份

## 结转相关凭证识别逻辑
后端会把摘要和备注拼接后做关键字匹配，当前关键字包括：
- 结转
- 损益
- 期末
- 汇兑
- 摊销
- 转账
- 结账
