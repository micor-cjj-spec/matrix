# 业务规则

## 查询规则
- `period` 格式建议为 `yyyy-MM`。
- 页面输入期间格式错误时，后端不会直接报错，而是回退到组织参数或系统当前月份，并返回 warning。
- 未选择业务单元时，仍可查询，但会提示“建议先选择业务单元后再检查”。

## 检查规则
后端当前会生成以下检查项：
- `ORG_CONFIG`：组织财务参数
- `ACCOUNTING_PERIOD`：会计期间
- `DEFAULT_VOUCHER_TYPE`：默认凭证字
- `FOUNDATION_HEALTH`：基础资料检查
- `CARRY_VOUCHERS`：结转相关凭证

## 任务状态规则
- `READY`：可执行
- `WARNING`：需处理
- `PENDING`：待准备
- `DONE`：已完成

## 识别规则
- 结转相关凭证不是独立表，而是从当前期间凭证中按摘要/备注关键字识别。
- 如果相关凭证全部已过账，则任务状态为 `DONE`；否则为 `WARNING`。

## 限制说明
- 当前凭证头没有组织维度，因此即使选择了业务单元，本期凭证数量统计仍按所选期间做全局统计。
