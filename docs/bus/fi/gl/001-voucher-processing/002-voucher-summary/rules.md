# 业务规则

## 查询规则
- `startDate` 和 `endDate` 必须满足开始日期不晚于结束日期。
- 日期格式必须为 `yyyy-MM-dd`，否则后端抛业务异常。
- `status` 会被后端统一转为大写后匹配。
- `summaryKeyword` 只对凭证头摘要做模糊匹配，不匹配备注和分录摘要。

## 统计规则
- 汇总金额基于凭证头 `famount` 聚合。
- 已过账金额只汇总状态为 `POSTED` 的凭证头金额。
- 页面“待处理凭证” = 已提交 + 已审核。
- 页面“异常凭证” = 已驳回 + 已冲销。

## 提示规则
- 当未传状态条件时，后端会返回一条提示，说明表格中的状态分布是所有状态一起汇总的。
- 当使用摘要关键字时，后端会返回一条提示，说明只匹配凭证头摘要。
- 无数据时，后端会返回“当前筛选条件下没有匹配的凭证”。
