# 流程设计

## 方案生成流程
1. 前端调用 `GET /arap-manage/plan`。
2. 后端按 `docTypeRoot` 决定源单据类型和结算单据类型。
3. 读取指定日期前的 AR/AP 单据，并扣减已落库核销链接得到未核销余额。
4. 按往来方分组，分别得到源单据和结算单据。
5. 采用“最早源单据优先匹配最早结算单据”的贪心规则生成方案。
6. 返回建议核销明细、剩余金额和提示信息。

## 匹配规则
- `AR`：源单据为 `AR / AR_ESTIMATE`，结算单据为 `AR_SETTLEMENT`
- `AP`：源单据为 `AP / AP_ESTIMATE`，结算单据为 `AP_PAYMENT_PROCESS`
- 匹配金额 = 源单据未核销余额 与 结算单据未分配余额 的较小值
