# 期初余额业务说明

## 1. 业务名称
期初余额（Opening Balance）

## 2. 业务背景
期初余额用于承接账簿在某一启用期间开始时的科目余额，是总账初始化和新期间滚动的重要起点。没有准确的期初余额，后续的发生额、余额和报表结果都无法保证正确。

## 3. 业务目标
- 为账簿启用提供初始核算数据
- 作为期间滚动和余额计算的起点
- 支撑历史数据导入和财务系统切换场景

## 4. 单据定位
期初余额属于总账模块的初始化业务数据，是余额体系的起始点，不是日常业务单据。

## 5. 核心业务内容
- 录入或导入科目期初借方、贷方、数量、外币等余额信息
- 校验科目方向、借贷平衡和辅助维度完整性
- 在启用账簿或新会计年度时生成初始余额基础

## 6. 上下游关系
### 上游
- 账簿
- 会计科目
- 会计期间

### 下游
- 科目余额计算
- 总账、明细账查询
- 财务报表
- 新年度余额结转

## 7. 关键控制点
- 期初余额导入后需校验借贷平衡
- 启用后通常不允许随意修改
- 涉及辅助核算时，应保证维度信息完整一致
