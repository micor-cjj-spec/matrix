# 应付单业务说明

## 1. 业务名称
应付单（Payable Bill）

## 2. 业务背景
应付单用于承接企业在采购、费用、服务等业务场景下形成的对外付款义务，是应付账款确认、结算和后续付款管理的核心业务单据。

## 3. 业务目标
- 统一沉淀应付业务来源和金额口径
- 支撑供应商结算、付款申请和账务处理
- 支撑应付余额、账龄和往来分析

## 4. 单据定位
应付单属于 `fi/ap` 域的核心往来业务单据，是付款与应付核算的主入口之一。

## 5. 核心业务内容
- 维护供应商、来源单据、应付金额、税额、到期日等信息
- 支撑审批、确认、结算、付款等后续动作
- 支撑与总账、资金等模块联动

## 6. 上下游关系
### 上游
- 采购单据或费用单据
- 供应商主数据
- 发票与结算资料

### 下游
- 付款申请
- 支付单
- 应付凭证

## 7. 关键控制点
- 应付金额口径必须与来源业务一致
- 已确认应付单的核心金额字段应限制修改
- 付款和核销需严格控制余额不超付
