# 往来方信用配置业务说明

## 1. 业务名称
往来方信用配置（Counterparty Credit）

## 2. 业务背景
往来方信用配置用于维护 AR/AP 维度的信用额度、逾期阈值及硬拦截规则。当前代码在单据提交、审核前会结合该配置做信用风险判断，并在命中硬拦截条件时直接阻断操作。fileciteturn165file0L12-L22 fileciteturn167file0L372-L413

## 3. 业务目标
- 为 AR/AP 往来单提供信用控制参数
- 支撑额度超限、逾期超阈值等预警与拦截
- 为账龄分析和信用预警提供配置基础

## 4. 单据定位
往来方信用配置属于 `fi/ar` 域下的规则配置对象，不直接形成业务单据流转，但会影响往来单提交与审核结果。fileciteturn166file0L31-L37

## 5. 核心业务内容
- 维护往来方、AR/AP 根类型、信用额度、逾期阈值、启用状态
- 维护“超额度硬拦截”“超逾期硬拦截”开关fileciteturn165file0L15-L22
- 为信用预警和业务拦截提供规则数据

## 6. 上下游关系
### 上游
- 往来方主数据
- 业务根类型（AR/AP）

### 下游
- 往来单提交与审核拦截
- 信用预警查询
- 账龄与风险分析

## 7. 关键控制点
- 信用额度必须大于 0。fileciteturn167file0L215-L224
- 逾期阈值为空或非法时会回落到默认值。fileciteturn167file0L221-L229
- 配置按 `counterparty + docTypeRoot` 唯一更新或新增。fileciteturn167file0L231-L254
