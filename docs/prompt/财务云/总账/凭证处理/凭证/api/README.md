# 凭证场景接口契约提示词

## 1. 对应 BUS 来源

- 业务域：财务云
- 模块：总账
- 子模块：凭证处理
- 场景：凭证
- 对应 BUS 目录：`docs/bus/财务云/总账/凭证处理/凭证/`

本提示词用于基于该 BUS 场景，生成“凭证”相关的接口清单、请求响应结构、分页与详情结构、错误码口径、状态流转类接口说明等接口契约文档，为前后端联调、后端实现、测试验证提供统一协议基础。

---

## 2. 生成目标

请基于 BUS 文档中关于“凭证”场景的业务目标、流程、规则、对象与状态定义，输出一份结构化接口契约文档，至少覆盖以下内容：

1. 接口边界与职责说明
2. 接口清单
3. 查询类接口契约
4. 操作类接口契约
5. 状态流转类接口契约
6. 请求参数与响应结构
7. 错误码与异常响应口径
8. 权限与幂等要求
9. 待确认接口项

输出结果应支撑以下后续工作：

- 前端页面与按钮动作对接
- 后端 Controller / DTO / VO 设计
- 测试用例与接口验证脚本设计
- Review 阶段契约一致性检查

- 与总账、账簿、组织、期间的关系

### 3.2 业务流程
重点识别以下动作是否存在：
- 新建凭证
- 保存凭证
- 查询凭证列表
- 查询凭证详情
- 修改凭证
- 删除凭证
- 提交凭证
- 审核凭证
- 反审核凭证
- 过账凭证
- 反过账凭证
- 作废凭证
- 冲销凭证
- 复制凭证
- 打印 / 导出凭证

### 3.3 业务规则
重点识别：
- 哪些状态可编辑
- 哪些状态可删除
- 哪些动作需要借贷平衡前置校验
- 哪些动作需要期间校验
- 哪些动作需要权限校验
- 哪些动作需要幂等控制

### 3.4 数据字典
生成 API 契约时，应优先复用并对齐：
- `docs/prompt/财务云/总账/凭证处理/凭证/dictionary/README.md` 中定义的字段与枚举口径

## 4. 生成约束

### 4.1 契约约束

- 不得编造 BUS 未定义的核心业务能力
- 接口命名、字段命名、状态命名必须与数据字典一致
- 对同一个业务动作，必须明确是“查询接口”还是“状态流转接口”
- 对批量动作与单笔动作必须区分

### 4.2 接口分类约束

输出中至少区分以下接口类型：

1. 列表查询接口
2. 详情查询接口
3. 新增 / 保存接口
4. 修改接口
5. 删除接口
6. 状态流转接口
7. 辅助查询接口
8. 导入导出 / 打印接口（若 BUS 存在）

### 4.3 请求结构约束

每个接口至少说明：

- 接口名称
- 接口用途
- 请求方法建议
- URI 建议
- 请求参数
- 返回结构
- 前置条件
- 权限要求
- 幂等要求
- 异常码
- 备注

### 4.4 响应结构约束

响应结构至少应覆盖：

- 通用响应包装
- 列表分页结构
- 详情对象结构
- 操作成功响应
- 操作失败响应

### 4.5 错误码约束

错误码必须按业务含义分类，例如：

- 参数校验失败
- 状态不允许操作
- 会计期间不允许操作
- 借贷不平衡
- 权限不足
- 数据不存在
- 重复提交 / 幂等冲突
- 系统异常

### 4.6 安全与权限约束

若 BUS 涉及权限控制，应显式给出：

- 查询权限
- 编辑权限
- 审核权限
- 过账权限
- 作废权限
- 冲销权限
- 导出 / 打印权限

对于状态变更类接口，应明确是否需要防重复提交与幂等控制。

## 6. 示例

### 6.1 示例接口清单

| 接口名称 | URI 建议 | 方法建议 | 类型 | 说明 |
|---|---|---|---|---|
| 查询凭证列表 | `/api/gl/vouchers` | GET | 查询 | 按编号、期间、状态等条件查询 |
| 查询凭证详情 | `/api/gl/vouchers/{id}` | GET | 查询 | 查询凭证头及分录明细 |
| 保存凭证 | `/api/gl/vouchers` | POST | 写操作 | 新增或保存草稿凭证 |
| 提交凭证 | `/api/gl/vouchers/{id}/submit` | POST | 状态流转 | 将草稿提交为待审核 |
| 审核凭证 | `/api/gl/vouchers/{id}/approve` | POST | 状态流转 | 审核通过后进入已审核 |
| 过账凭证 | `/api/gl/vouchers/{id}/post` | POST | 状态流转 | 将已审核凭证过账 |

### 6.2 示例错误码

| 错误码 | 含义 | 触发条件 |
|---|---|---|
| GL_VOUCHER_001 | 凭证不存在 | 根据 ID 未查询到凭证 |
| GL_VOUCHER_002 | 当前状态不允许提交 | 非草稿状态下提交 |
| GL_VOUCHER_003 | 借贷不平衡 | 提交前校验借贷不等 |
| GL_VOUCHER_004 | 会计期间未打开 | 当前期间不允许操作 |
| GL_VOUCHER_005 | 无审核权限 | 当前用户无审核权限 |

## 7. 验收标准

生成结果满足以下要求才视为合格：

1. 能覆盖凭证场景主要查询、写操作和状态流转能力
2. 接口字段、状态、枚举与数据字典保持一致
3. 能明确区分前置条件、权限要求与幂等要求
4. 能支撑前后端联调与测试设计，不产生明显契约歧义
5. 对 BUS 未明确的接口项有显式标记，而非擅自补齐
