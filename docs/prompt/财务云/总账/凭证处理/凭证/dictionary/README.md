# 凭证场景数据字典提示词

## 1. 对应 BUS 来源

- 业务域：财务云
- 模块：总账
- 子模块：凭证处理
- 场景：凭证
- 对应 BUS 目录：`docs/bus/财务云/总账/凭证处理/凭证/`

本提示词用于基于该 BUS 场景，生成“凭证”相关的数据字典、字段口径、枚举定义、状态机说明与关键校验语义，作为后续接口、后端、前端、SQL、测试生成的统一语义底座。

---

## 2. 生成目标

请基于 BUS 文档中关于“凭证”场景的业务描述、流程、规则与对象定义，输出一份可直接用于研发和测试协同的数据语义文档，至少覆盖以下内容：

1. 核心业务对象定义
2. 核心字段字典
3. 枚举与状态定义
4. 状态流转说明
5. 关键业务校验口径
6. 关键计算口径
7. 与上下游对象的关系说明
8. 命名口径与展示口径

输出结果应支撑以下后续工作：

- API 请求/响应字段统一
- 后端实体、DTO、VO、枚举统一
- 前端表单与列表字段统一
- SQL 字段命名与注释统一
- 测试用例校验口径统一

## 3. 输入上下文

生成时应综合消费以下输入：

### 3.1 BUS 场景总说明
重点识别：
- 凭证场景的业务目标
- 凭证在总账中的定位
- 凭证与单据、分录、账簿、组织、期间的关系

### 3.2 业务流程与状态流转
重点识别：
- 凭证创建
- 凭证保存
- 凭证提交
- 凭证审核
- 凭证过账
- 凭证作废
- 凭证冲销
- 凭证反审核/反过账（若 BUS 有定义）

### 3.3 业务规则
重点识别：
- 借贷平衡规则
- 会计期间校验
- 组织 / 账簿 / 币别约束
- 科目启用状态约束
- 分录必填字段约束
- 凭证编号规则
- 作废、冲销、反操作限制

### 3.4 业务对象
重点识别：
- 凭证头
- 凭证明细 / 分录
- 会计科目
- 核算维度
- 账簿
- 组织
- 会计期间
- 来源单据
- 审核人与过账人

## 4. 生成约束

### 4.1 语义约束

- 不得编造 BUS 中不存在的业务动作、字段或规则
- 若 BUS 未明确某一字段或状态，必须标记为“待 BUS 明确”
- 字段说明必须区分“业务含义”和“技术含义”
- 相同概念必须保持唯一命名，不得同义混用

### 4.2 结构约束

输出必须至少包含以下章节：

1. 场景概述
2. 核心业务对象
3. 核心字段字典
4. 枚举定义
5. 状态机说明
6. 关键校验规则
7. 关键计算口径
8. 对外展示口径
9. 待确认项

### 4.3 字段字典约束

核心字段字典至少应包含以下列：

- 所属对象
- 字段中文名
- 字段英文名建议
- 数据类型建议
- 是否必填
- 业务含义
- 来源 / 去向
- 示例值
- 备注

### 4.4 枚举与状态约束

至少梳理以下枚举或状态（若 BUS 有定义）：

- 凭证状态
- 凭证来源类型
- 凭证字类型
- 借贷方向
- 分录状态
- 过账状态
- 作废状态
- 冲销关系类型

### 4.5 安全与权限约束

若 BUS 中涉及安全与权限，应一并抽取并体现在字典说明中，包括但不限于：

- 哪些动作需要审核权限
- 哪些动作需要过账权限
- 哪些字段仅特定角色可见
- 哪些状态下不允许修改

## 6. 示例

### 6.1 示例字段字典

| 所属对象 | 字段中文名 | 字段英文名建议 | 数据类型建议 | 是否必填 | 业务含义 | 来源/去向 | 示例值 | 备注 |

## 7. 验收标准

生成结果满足以下要求才视为合格：

1. 能清晰区分凭证头、分录及关联对象的字段语义
2. 能覆盖凭证场景关键状态、关键枚举和关键规则
3. 能支撑后续 API、后端、前端、SQL、测试生成，不产生明显歧义
4. 对 BUS 未明确内容有显式标记，而不是主观补全
5. 命名统一、表述清晰、便于研发直接使用
