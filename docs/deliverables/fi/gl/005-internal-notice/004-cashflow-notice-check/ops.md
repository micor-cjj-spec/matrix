# 现金流通知单勾稽运维说明

## 1. 发布关注点
- 前端 `CashflowNoticeCheckView.vue` 与后端 `/internal-notice/cashflow/reconcile` 接口保持兼容
- 最新现金流候选结果依赖需保持可用
- 跳转到凭证页规则需保持一致

## 2. 监控关注点
- 现金流通知单勾稽接口响应耗时
- 现金流通知单勾稽接口错误率
- warnings 返回异常增长情况
- 跳转凭证页联动问题

## 3. 风险与回滚
- 若勾稽口径异常，优先回滚到已验证的比对逻辑
- 若页面展示异常但接口正常，可先回退前端展示变更
- 若跳转不兼容，先保留勾稽查询能力，降级外部联动能力

## 4. 说明
- 当前场景不涉及执行类落库操作，不需要执行类运维方案
