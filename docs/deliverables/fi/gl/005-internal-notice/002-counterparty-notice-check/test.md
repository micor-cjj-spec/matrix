# 往来通知单勾稽测试说明

## 1. 核心验证范围
- 查询场景
- 通知状态、统计日期过滤
- 汇总卡片展示
- warnings 展示
- 勾稽结果表展示
- 跳转到往来对账单并决定 `openOnly=true`

## 2. 关键测试点
1. 点击查询后能正确展示勾稽结果
2. 不同过滤条件能正确作用于结果
3. 仍需处理、已自然解除、快照未清、当前未清展示正确
4. warnings 展示正确
5. 跳转到往来对账单时 `openOnly` 参数正确
6. 空结果、异常结果、依赖加载失败能正确展示

## 3. 风险点
- 快照未清与当前未清口径不一致
- 勾稽状态判断不一致
- `openOnly` 跳转规则不一致
