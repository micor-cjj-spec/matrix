# 现金流通知单运维说明

## 1. 发布关注点
- 前端 `CashflowNoticeView.vue` 与 `/internal-notice/cashflow`、`/internal-notice/cashflow/generate` 接口保持兼容
- 现金流补充资料待处理凭证与内部通知单表依赖需保持可用
- 跳转到凭证页与现金流通知单勾稽页面参数映射需保持一致

## 2. 监控关注点
- 现金流通知单查询接口响应耗时
- 现金流通知单生成接口响应耗时与错误率
- warnings 返回异常增长情况
- 自动关闭通知数量异常波动

## 3. 风险与回滚
- 若通知生成或查询口径异常，优先回滚到已验证的逻辑
- 若页面展示异常但接口正常，可先回退前端展示变更
- 若跳转不兼容，先保留通知查询与生成能力，降级外部联动能力

## 4. 说明
- 当前场景包含真实生成能力，发布需重点关注幂等与生成统计口径
