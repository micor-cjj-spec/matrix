# 往来通知单评审清单

## 1. 一致性检查
- BUS 与 prompt 是否一致
- 通知状态、紧急程度、通知金额、未清金额、生成结果统计口径是否一致
- 查询、生成、成功提示区、跳转到对账单/勾稽页面是否一致
- 往来类型、通知状态、紧急程度、统计日期过滤口径是否一致

## 2. 已对齐项
- 页面定位为把账龄与风险扫描结果转化为内部处理单的页面
- 查询接口为 `GET /internal-notice/counterparty`
- 生成接口为 `POST /internal-notice/counterparty/generate`
- 页面为 `CounterpartyNoticeView.vue`
- 数据来源为往来账龄分析结果和内部通知单表

## 3. 待确认项
- 通知状态枚举是否需要独立文档化
- 自动关闭触发条件是否需要更细说明
