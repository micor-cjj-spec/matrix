# 往来核销日志口径说明

## 1. 查询条件口径
- `docTypeRoot`：往来类型
- `counterparty`：往来方
- `planCode`：方案号
- `startDate`：开始日期
- `endDate`：结束日期

## 2. 汇总统计口径
- `logCount`：批次数
- `linkCount`：链接数
- `totalAmount`：核销金额
- `明细查看状态`：当前是否已按方案号下钻明细

## 3. 页面区域口径
- `records`：批次表，展示方案号、操作人、操作时间、核销金额、链接数等摘要
- `linkDetails`：明细表，展示指定方案号下的源单据/结算单据配对明细
- `warnings`：需要重点关注的提示信息

## 4. 联动口径
- 点击批次行“查看明细”后，把该批次 `planCode` 带回查询并刷新明细表

## 5. 待确认项
- 明细查看状态的最终展示规则是否需要更细说明
- warnings 的正式触发规则是否需要单独固化
