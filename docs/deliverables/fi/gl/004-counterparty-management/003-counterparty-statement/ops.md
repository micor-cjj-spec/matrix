# 往来对账单运维说明

## 1. 发布关注点
- 前端 `CounterpartyStatementView.vue` 与后端 `/arap-manage/statement` 接口保持兼容
- 查询依赖的往来方、核销日志、核销链接数据需保持可用
- 关联凭证跳转与核销日志跳转参数映射需保持一致

## 2. 监控关注点
- 往来对账单接口响应耗时
- 往来对账单接口错误率
- warnings 返回异常增长情况
- 最近批次摘要查询异常率

## 3. 风险与回滚
- 若对账口径异常，优先回滚到已验证的查询逻辑
- 若页面展示异常但接口正常，可先回退前端展示变更
- 若跳转参数不兼容，先保留对账查询能力，降级跳转能力

## 4. 说明
- 当前场景不涉及执行类落库操作，不需要核销执行类运维方案
