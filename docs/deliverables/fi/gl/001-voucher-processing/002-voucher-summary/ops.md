# 凭证汇总表运维说明

## 1. 发布关注点
- 前端 `VoucherSummaryView.vue` 与后端 `/voucher/summary` 接口需保持兼容
- 若统计口径有调整，需要同步验证 warnings 与表格字段

## 2. 监控关注点
- 汇总接口响应耗时
- 汇总接口错误率
- warnings 返回异常增长情况
- 跳转到凭证列表的兼容性问题

## 3. 风险与回滚
- 若统计结果异常，优先回滚到已验证的聚合逻辑
- 若页面展示异常但接口正常，可先保留 API 能力并回退前端展示变更
- 若跳转参数不兼容，先关闭或弱化跳转入口，保留汇总浏览能力

## 4. 说明
- 当前场景不涉及直接修改凭证，不需要制单类操作运维方案
- 当前页面不支持导出，不应增加不存在的发布依赖
