# 凭证汇总表测试说明

## 1. 核心验证范围
- 默认本月区间初始化查询
- 日期区间、状态、摘要关键字查询
- 汇总卡片统计结果展示
- warnings 展示
- 按日期表格行展示
- 查看凭证跳转

## 2. 关键测试点
1. 查询条件变化后统计结果应正确刷新
2. 重置后应恢复默认日期并重新查询
3. 空结果时页面应正确展示空态
4. 接口异常时页面应正确展示异常态
5. warnings 返回时应正确展示告警信息
6. 点击“查看凭证”应带着当天日期、状态、摘要条件跳转

## 3. 重点风险
- 统计口径与后端实现不一致
- warnings 展示逻辑与后端返回不一致
- 跳转参数与凭证列表页过滤口径不一致

## 4. 建议
- 将统计结果校验拆分为 API 层与页面层双重验证
- 对默认本月查询与跳转参数做回归检查
