# 结转清单评审清单

## 1. 一致性检查
- BUS 中“检查与导航”定位是否与 prompt 一致
- 检查项、warnings、相关凭证口径是否一致
- 页面展示、接口返回、查看凭证跳转是否一致
- 业务单元与期间口径是否一致

## 2. 已对齐项
- 页面只做检查与导航，不直接生成结转凭证
- 接口为 `GET /voucher/carry-list`
- 页面为 `CarryForwardListView.vue`
- 页面初始化依赖业务单元列表

## 3. 待确认项
- 基础资料健康度的最终计算规则
- 检查完成度或任务状态的最终展示口径

## 4. 后续建议
- 若后续增加执行类动作，应单独新增 BUS、prompt 与交付物
- 若相关凭证识别逻辑调整，应同步更新测试与样例说明
