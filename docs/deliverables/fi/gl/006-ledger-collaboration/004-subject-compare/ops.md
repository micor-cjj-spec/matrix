# 科目余额对照运维说明

## 1. 发布关注点
- 前端 `SubjectCompareView.vue` 与后端 `/ledger-collaboration/subject-balance-compare` 接口保持兼容
- 科目选项与分录汇总依赖需保持可用
- 跳转到科目余额表页面参数映射需保持一致

## 2. 监控关注点
- 科目余额对照接口响应耗时
- 科目余额对照接口错误率
- warnings 返回异常增长情况
- 借贷差值与余额差额统计异常波动

## 3. 风险与回滚
- 若对照逻辑或差异口径异常，优先回滚到已验证的查询逻辑
- 若页面展示异常但接口正常，可先回退前端展示变更
- 若跳转不兼容，先保留对照查询能力，降级外部联动能力

## 4. 说明
- 当前场景不涉及执行类落库操作，不需要执行类运维方案
