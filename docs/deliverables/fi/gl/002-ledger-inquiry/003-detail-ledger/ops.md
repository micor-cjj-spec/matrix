# 明细分类账运维说明

## 1. 发布关注点
- 前端 `DetailLedgerView.vue` 与后端 `/ledger/detail-ledger` 接口保持兼容
- 初始化自动查询逻辑与页面参数保持一致
- 科目选项加载依赖需保持可用

## 2. 监控关注点
- 明细分类账接口响应耗时
- 明细分类账接口错误率
- warnings 返回异常增长情况
- 科目选项加载失败率

## 3. 风险与回滚
- 若流水或余额计算异常，优先回滚到已验证的查询逻辑
- 若页面展示异常但接口正常，可先回退前端展示变更
- 若初始化自动查询行为不兼容，先保留手动查询能力并降级自动触发

## 4. 说明
- 当前场景不涉及写操作，不需要制单类运维方案
