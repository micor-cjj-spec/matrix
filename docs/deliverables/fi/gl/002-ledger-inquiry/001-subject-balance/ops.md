# 科目余额表运维说明

## 1. 发布关注点
- 前端 `SubjectBalanceView.vue` 与后端 `/ledger/subject-balance` 接口保持兼容
- 路由 query 参数与页面查询参数口径保持一致
- 科目下拉共享加载能力需保持可用

## 2. 监控关注点
- 余额表接口响应耗时
- 余额表接口错误率
- warnings 返回异常增长情况
- 科目下拉加载失败率

## 3. 风险与回滚
- 若余额计算异常，优先回滚到已验证的查询逻辑
- 若页面展示异常但接口正常，可先回退前端展示变更
- 若 query 参数不兼容，先保留基础查询能力，降级路径传参能力

## 4. 说明
- 当前场景不涉及写操作，不需要制单类运维方案
