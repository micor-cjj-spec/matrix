# 报表项目运维说明

## 1. 发布关注点
- 前端 `ReportItemView.vue` 与 `/report-item/*` 接口保持兼容
- 报表模板与报表项目基础数据依赖需保持可用
- 当前前端未接入新增、编辑、删除入口，发布时需避免误开放

## 2. 监控关注点
- `GET /report-item/list` 响应耗时与错误率
- 模板加载接口响应耗时与错误率
- tree/detail/write 接口的异常访问情况
- 报表项目基础数据缺失或层级异常情况

## 3. 风险与回滚
- 若列表查询或模板加载异常，优先回滚到已验证的查询逻辑
- 若层级或排序展示异常，可先保留基础分页列表能力
- 若后续开放写入口出现兼容问题，应先关闭入口并保留后端能力

## 4. 说明
- 当前场景以查询展示为主
- 写接口虽存在，但当前前端未接入，运维重点仍应放在查询链路与基础数据完整性
