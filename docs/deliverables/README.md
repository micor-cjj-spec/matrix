# Deliverables 目录说明

## 1. 目录目标

`docs/deliverables/` 用于沉淀非代码类正式交付物。

这里存放的是经过确认、可审阅、可执行、可交接的非代码资产，不存放前后端正式源码。

---

## 2. 与其他目录的关系

- `docs/bus/`：定义业务场景
- `docs/prompt/`：定义生成交付物的提示词
- `docs/deliverables/`：保存非代码正式交付物
- `matrix`：后端正式代码
- `matrix-web`：前端正式代码

推荐追溯链路：

`docs/bus/{scene} -> docs/prompt/{scene} -> docs/deliverables/{scene}`

前后端代码单独落在对应工程中。

---

## 3. 目录结构

```text
docs/deliverables/
└── {scene}/
    ├── README.md
    ├── sql/
    ├── test/
    ├── ops/
    ├── api/
    ├── dictionary/
    ├── mock/
    └── review/
```

---

## 4. 场景目录说明

每个业务场景在 `docs/deliverables/` 下建立同名目录，并使用 `README.md` 作为该场景交付物总说明。

`README.md` 建议包含：

1. 对应 BUS 场景
2. 对应 Prompt 来源
3. 交付物清单
4. 使用顺序
5. 环境要求
6. 风险与注意事项
7. 回滚与兜底说明

---

## 5. 各类交付物说明

### 5.1 sql

用于保存正式 SQL 产物，例如：

- 建表脚本
- 变更脚本
- 初始化数据脚本
- 数据修复脚本

### 5.2 test

用于保存正式测试验证产物，例如：

- 测试用例
- 回归检查清单
- 测试报告
- 接口测试脚本
- UAT 验收清单

### 5.3 ops

用于保存正式运维产物，例如：

- 部署文档
- 环境配置说明
- 回滚预案
- 监控检查清单
- 应急处置手册

### 5.4 api

用于保存正式接口契约产物，例如：

- 接口清单
- 请求响应说明
- 错误码说明
- OpenAPI 草稿

### 5.5 dictionary

用于保存正式数据语义产物，例如：

- 字段字典
- 枚举定义
- 状态机说明
- 校验规则说明

### 5.6 mock

用于保存正式样例产物，例如：

- 请求样例
- 响应样例
- 演示数据
- 边界场景样例

### 5.7 review

用于保存正式检查类产物，例如：

- 代码评审清单
- 测试评审清单
- 上线检查清单
- 一致性检查清单

---

## 6. 代码落点说明

- 前端正式代码：`matrix-web`
- 后端正式代码：`matrix`
- 非代码正式交付物：`docs/deliverables/{scene}`
