# 总账-余额表业务设计（初版）

> 模块：GL（总账）
> 主题：余额表（Balance Table）
> 更新时间：2026-03-12

## 1. 业务目标
余额表用于沉淀“按期间、按科目、按维度”的期初、本期借贷发生、期末余额，是总账查询、科目余额表、试算平衡和报表取数的核心中间层。

## 2. 建议定位
放在**基础设置 / 总账设置**下作为“数据基础能力”，但数据来源必须是“已过账分录（GL Entry）”，禁止手工改余额。

## 3. 数据来源与生成策略
- 唯一来源：`bizfi_fi_gl_entry`（凭证过账后写入）
- 生成方式建议：
  1. **实时增量**：每次凭证过账/冲销时增量更新余额表
  2. **定时重算任务**：每天夜间重算兜底，防止历史异常

> 实际落地建议：先做“日终重算 + 手工重建按钮”，稳定后再上实时增量。

## 4. 余额表核心口径
按 `公司/账套 + 会计期间 + 科目 + 维度组合 + 币种` 聚合：
- 期初借方余额
- 期初贷方余额
- 本期借方发生
- 本期贷方发生
- 期末借方余额
- 期末贷方余额

并遵循“方向余额制”：同一时点只保留借或贷一侧余额（另一侧为0）。

## 5. 核心字段建议
- `fcompany_id` 公司/账套
- `fperiod` 会计期间（yyyy-MM）
- `faccount_code` 科目编码
- `faux_key` 辅助核算键（客户/供应商/部门/项目等拼接）
- `fcurrency` 币种
- `fopen_debit`, `fopen_credit`
- `fperiod_debit`, `fperiod_credit`
- `fclose_debit`, `fclose_credit`
- `fversion` 重算版本号（便于审计和幂等）
- `fupdated_time`

## 6. 关键业务规则
1. 余额表只读，禁止前台手工改金额。
2. 只统计 `POSTED` 分录；冲销分录按正常分录参与计算。
3. 关闭期间不可重算（除管理员“解锁重算”）。
4. 凭证反过账/异常补记后，必须触发余额重建。
5. 科目停用不影响历史期间余额查询。

## 7. 页面建议（基础设置-余额表）
- 查询条件：期间、科目范围、是否含辅助核算、币种
- 功能按钮：
  - 查询
  - 导出Excel
  - 余额重建（按期间）
  - 余额校验（与分录对账）
- 列展示：期初 / 本期借贷发生 / 期末余额 / 差异标识

## 8. 与现有凭证能力的关系
- 过账写 `bizfi_fi_gl_entry`
- 余额表从 `bizfi_fi_gl_entry` 汇总
- 报表从余额表读取（性能更稳）

## 9. 分期落地建议
### Phase 1（本周）
- 建余额表表结构
- 做“按期间重算接口”
- 前端做查询+导出

### Phase 2
- 过账后增量更新余额
- 增加异常重算队列和审计日志

### Phase 3
- 多币种并行余额、辅助核算余额、对账中心
